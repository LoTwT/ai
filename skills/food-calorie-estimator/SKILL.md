# 食物热量估算方法论

## 核心原则

三条不可违反的规则：

1. **每个数字必须有推导过程** — 绝不凭直觉直接给出一个总热量数字
2. **必须先有可量化份量，再算热量** — 固体先估重量，饮品先估容量；只有包装/品牌已给出精确总热量时才可直接使用
3. **不确定时取高值** — 贯穿所有阶段：
   - 不确定烹饪方式 → 取修正倍数高档
   - 不确定份量 → 取合理范围偏高值
   - 不确定食物品种 → 取同类中热量较高的
   - 不确定饮品糖度（未标注或看不清） → 按全糖算
   - 不确定饮品加料（有无珍珠/奶盖等） → 须追问（见追问决策矩阵）
   - 不确定杯型 → 按大杯算

### 数据获取优先级

本 skill 不硬编码具体菜品的热量数值（菜品热量应实时查询）。容器尺寸、食物密度、饮品组件等结构性物理常量可以固定，因为它们不随烹饪方式变化。所有菜品热量数据应在估算时通过以下优先级获取：

1. **包装营养标签**（照片中可见） → 直接读取，最准确
2. **品牌官方数据**（识别到品牌 logo/包装） → Web 搜索品牌官方确认
3. **训练知识**（中国食物成分表、USDA 等） → 按推理流程计算
4. **Web 搜索**（以上都不确定时） → 搜索交叉验证

### 交叉验证规则

对训练知识给出的数据，如果是高热量食物（>200kcal/100g）或不常见食物，主动搜索一次验证。搜索结果与训练知识偏差 >20% 时，取高值并标注不确定。

## 第一阶段：食物识别

### 扫描顺序

拿到一张食物照片后，先执行零号步骤，再按顺序系统扫描：

**零号步骤：品牌预判** — 快速扫描照片中是否有明确的品牌标识（logo、杯贴、包装文字）。如果有，先为该品牌项走下文"连锁品牌快捷路径"，然后继续系统扫描处理照片中的其余食物。没有品牌标识时，直接按以下顺序扫描，避免遗漏：

1. **主食**（米饭、面、馒头、饼、年糕） — 通常体积最大、颜色浅
2. **蛋白质主菜**（肉、鱼、蛋、豆制品） — 盘中最显眼的部分
3. **蔬菜/配菜** — 绿色或彩色部分
4. **汤类** — 碗中有汤水的食物，注意辨认配料
5. **饮品** — 牛奶盒、咖啡杯、饮料瓶罐、奶茶杯
6. **零食/包装食品** — 独立包装物

### 强制三元组输出

每识别一项食物，必须产出以下三元组才能进入下一步：

```
(食物名称, 烹饪方式, 识别依据)
```

示例：`(鸡腿, 红烧, 表面深色酱汁裹覆+有光泽)`

三元组中任一项为"不确定"时，触发追问决策矩阵。无烹饪方式的食物（水果、包装食品等），烹饪方式填"N/A"。

### 识别关键点

**看文字**：包装上的品牌名、品名、口味、容量、营养标签都是线索。饮品杯上的贴纸通常包含名称、杯型、甜度、定制选项。

**看烹饪方式**：同一食材不同做法热量差异极大，从外观判断：
- 表面油亮、有焦痕 → 煎/炸/炒
- 颜色清淡、有汤水 → 蒸/煮/烫/炖
- 深色酱汁裹覆 → 红烧/卤/酱
- 表面干燥、有烤痕 → 烤/烘

### 追问决策矩阵

不是所有不确定都需要问用户。按热量影响决定：

| 情况 | 热量影响 | 行为 |
|---|---|---|
| 肉类品种不确定（猪/牛/鸡/鸭） | 可达 2 倍差异 | **必须问** |
| 烹饪方式不确定（炒 vs 炸） | 可达 1.5 倍差异 | **必须问** |
| 饮品加料不确定（有无珍珠/奶盖） | 可达 2 倍差异 | **必须问** |
| 饮品糖度不确定（全糖/半糖） | ~20% 差异 | 不问，按全糖算 |
| 蔬菜品种不确定（菠菜 vs 油菜） | <10% 差异 | 不问，取高值 |
| 调味差异（咸/淡） | 几乎无差异 | 不问 |
| 份量大小不确定 | 取决于偏差 | 可能偏差 >30% 才问 |

核心逻辑：**热量影响大就问，影响小就取高值不打扰用户。**

### 连锁品牌快捷路径

识别到品牌 logo、包装、杯贴时：
1. 确认品牌和具体产品名
2. Web 搜索该品牌官方公布的热量数据
3. 找到且规格可确认（如杯型/容量/版本明确） → 直接使用（品牌 SKU 数据是精确产品定义，规格明确时可直接作为最终热量），跳过后续估算
4. 找不到，或规格无法确认 → 回到常规估算流程，并对不确定项取高值

## 第二阶段：重量估算

这是最关键的一步。**固体食物永远不要跳过重量直接给热量**；饮品则先估容量，再按后文的 ml 流程计算。只有包装/品牌已给出精确总热量时，才允许直接使用。

### 三级尺寸锚定

> **范围取值规则**：以下参照表中给出范围时（如容量 500-650ml、密度 0.6-0.7），如无法从照片判断更精确的值，一律取范围上限。

#### 第一级：容器锚定（主要方法）

几乎所有食物照片都有容器（碗、盘、外卖盒、杯子），容器本身就是最可靠的尺寸参照。

**步骤**：
1. 判断容器类型（饭碗/汤碗/盘子/外卖盒）
2. 查容器参照表确定容量
3. 观察填充程度（满/八分满/半碗/浅浅一层）
4. 容量 × 填充比例 × 食物密度 = 重量

**家用餐具参照**：

| 容器 | 口径 | 容量 | 识别特征 |
|---|---|---|---|
| 标准饭碗 | 11-12cm | 300-400ml | 最常见，单手可握 |
| 大饭碗/汤碗 | 15-18cm | 600-800ml | 双手端 |
| 小碟 | 10-12cm | — | 酱料/小菜 |
| 家常菜盘 | 20-23cm | 面积约 300-400cm² | 圆形平盘 |
| 大盘 | 25-30cm | 面积约 500-700cm² | 多人菜 |

**外卖容器参照**：

| 容器 | 尺寸 | 容量 | 识别特征 |
|---|---|---|---|
| 小圆盒 | 口径 12cm | 300-400ml | 装米饭/小份菜 |
| 标准方盒 | 15×12cm | 500-650ml | 最常见外卖盒 |
| 大方盒 | 18×13cm | 750-1000ml | 大份/双拼 |
| 长方盒（便当） | 22×15cm | 800-1000ml | 分格便当 |
| 圆形汤碗 | 口径 14-16cm | 400-500ml | 带盖 |

**餐厅餐具参照**：

| 容器 | 口径/容量 | 识别特征 |
|---|---|---|
| 米饭小碗 | 10-11cm / 约 250ml | 比家用碗小一号 |
| 面碗 | 18-20cm / 约 800-1000ml | 高且深 |
| 砂锅 | 16-20cm | 厚壁、带盖 |

**食物密度表**：

|---|---|---|
| 米饭（松散） | 0.6-0.7 | 家常盛饭 |

| 叶菜（炒/烫） | 0.3-0.5 | 疏松 |
| 根茎类（土豆/萝卜） | 0.8-1.0 | 密实 |
| 豆腐 | 0.9-1.0 | 接近水 |
| 猪肉（块/片） | 0.8-1.0 | 密实 |
| 鸡肉（块） | 0.7-0.9 | 带骨偏低 |

| 虾仁 | 0.7-0.8 | 有间隙 |

| 坚果 | 0.5-0.7 | 有间隙 |
| 汤类 | ~1.0 | 接近水 |

#### 第二级：已知食物锚定（补充校准）

如果照片中存在尺寸固定的食物，用它校准其他食物的大小：

| 锚定食物 | 标准尺寸 |
|---|---|
| 鸡蛋 | 长约 5.5cm，去壳约 50g |
| 饺子 | 长约 7-8cm，约 25-30g/个 |
| 馒头（标准） | 直径约 8cm，约 80-100g |
| 易拉罐 | 高 12.3cm，直径 6.6cm |
| 矿泉水瓶 | 高 21cm，直径 6.5cm |

这些不是要求用户额外放置的参照物，而是它们本身就是食物或常见桌面物品，照片中碰到了就用来校准。

#### 第三级：标准份量兜底（无容器时）

极少数情况下容器不可见（手持食物、特写照片），用标准份量默认值。一律取范围高值，符合保守原则：

| 场景 | 默认份量（取高值） |
|---|---|
| 一人份米饭 | 200g |
| 一人份面条（熟） | 350g |
| 餐厅一份炒菜 | 300g |
| 一个肉夹馍/煎饼 | 250g |
| 一根玉米 | 200g（含芯） |
| 一个苹果/梨 | 250g |
| 一根香蕉 | 150g（含皮） |

### 可食用比例折算

带骨、带壳、带皮的食物，估算重量后需折算可食用部分：

| 食物 | 可食用比例 |
|---|---|
| 带骨鸡（鸡腿/鸡翅） | ×0.7 |
| 排骨 | ×0.6 |
| 带壳虾 | ×0.5 |
| 带壳蟹 | ×0.4 |
| 带皮水果（橙/柚） | ×0.7 |

不确定时取可食用比例的高值（即少打折，多算可食用重量，从而多算热量）。

### 辅助方法

#### 数量计数法

可以数个数的食物，逐个计数再查单个重量。

1. 数清个数
2. 回忆或搜索该食物的单个标准重量（如鸡蛋去壳约 50g、鹌鹑蛋去壳约 8g、馄饨约 25-35g/个）
3. 个数 × 单个重量 = 总重量

这是最准确的估算方法之一，优先用于可数食物。

#### 面积厚度法

盘中平铺的食物（如切片肉、炒菜），用面积 × 厚度估算体积。

1. 估算食物覆盖的面积（参照盘子大小）
2. 估算平均厚度（一层叶菜约 1-2cm，肉片堆约 2-3cm）
3. 面积 × 厚度 = 体积
4. 体积 × 密度 = 重量

#### 包装直接读取

包装食品上有净含量，直接使用，不需要估算。找到"净含量"标注（如"50g"、"250ml"）。对于液体，ml 和 g 可近似 1:1。

#### 份量比例法

同一食物在不同餐次出现时，通过与之前的视觉对比估算份量。

1. 回忆之前同一食物的照片和估算重量
2. 对比当前照片中的盛装量
3. 按比例缩放重量

这是保持一致性的关键方法，详见第四阶段。

### 同一碗/盘多种食物的拆分

一个容器中有多种食物时：
1. 先估计整个容器的食物总体积
2. 目测每种食物占据的体积比例（注意堆叠情况）
3. 分别按各自密度计算重量

## 第三阶段：热量计算

### 固体食物常规计算路径

以下公式适用于无法直接读取总热量的固体食物。饮品按后文的 ml 流程处理；包装营养标签和规格明确的品牌 SKU 可直接使用总热量；汤类和复合菜按后文特殊处理。

```
热量(kcal) = 估算重量(g) × 成品每100g热量(kcal) ÷ 100
```

在该路径下，优先使用**成品每100g热量**数据。成品数据已经包含了烹饪方式、调味、用油量的影响，不需要任何额外修正。

### 数据获取流程

```
第一步：回忆训练知识中的成品数据（中国食物成分表、USDA）
         ↓ 确信 → 乘以估算重量，完成
         ↓ 不确信 / 高热量食物(>200kcal/100g) / 不常见食物
第二步：Web 搜索 "[菜品名] 每100g 热量" 交叉验证或获取数据
         ↓ 找到 → 与训练知识对比，取高值，完成
         ↓ 没找到
第三步：换同义词/近义菜名再搜
         ↓ 找到 → 同上
         ↓ 没找到
第四步（最后手段）：拆解为原料，逐个查原料每100g数据，加三档烹饪修正
         ↓
可选校验：搜 "[菜品名] 一份 热量" 对比结果是否合理
```

**关键规则**：
- 成品数据**不需要任何烹饪修正**，因为已经包含了烹饪方式的影响
- "一份"数据**只用于校验**，不作为计算输入 — 因为"一份"的份量定义不明确，无法与照片中的实际重量对应。**例外**：品牌 SKU 数据（如"瑞幸生椰拿铁 大杯 267kcal"）是精确产品定义，可直接使用
- 中国标签常用"千焦(kJ)"，需除以 4.184 转换为 kcal

### 三档烹饪修正（仅用于查不到成品数据、只有原料数据时）

| 档位 | 烹饪方式 | 统一修正 |
|---|---|---|
| 低油 | 水煮、蒸、烫、白灼、凉拌 | ×1.0 |
| 中油 | 炒、红烧、卤、糖醋 | ×1.3 |
| 高油 | 煎、油炸、干炸、干煸、干锅、拔丝 | ×1.8 |

不确定属于哪档 → 上提一档。

### 汤类特殊处理

汤的热量不能简单按总重量算，因为大部分重量是水（几乎 0 热量）。

**正确做法**：
1. 分别识别汤中的固体配料（肉、蛋、豆腐、蔬菜等）
2. 估算每种固体配料的重量
3. 分别查每种配料的热量
4. 汤水部分粗估：清汤 5-15kcal/100ml，浓汤/奶汤 20-40kcal/100ml
5. 固体热量 + 汤水热量 = 总热量

### 复合菜品处理

多种食材混合的菜品（如咖喱鸡肉配土豆胡萝卜）：

- **方法 A（整体法）**：直接搜索该菜品的成品热量数据 → 乘以估算重量（优先使用）
- **方法 B（拆分法）**：估算总重量 → 目测各食材比例 → 分别算热量 → 求和（整体法搜不到时使用）

### 额外蘸料处理

只处理照片中**明确可见的蘸碟**，不猜测看不到的调味：

| 蘸料 | 热量/份 |
|---|---|
| 酱油+醋 | 忽略不计 |
| 麻酱/花生酱 | 80-100kcal |
| 辣椒油 | 40-60kcal |
| 沙拉酱 | 50-80kcal |

### 饮品热量计算（独立流程）

饮品不走上述计算路径。先判断走哪条分支：

```
         识别饮品
            ↓
    是否为已知品牌/产品？
     ↓ 是                       ↓ 否
  品牌快捷路径               是否可拆解为基底+添加物？
  （直接用 SKU 数据）         ↓ 是                     ↓ 否
                          基底+添加物流程          搜索 "[饮品名] 热量"
                          （下方四步流程）          按同类高值估算
```

以下拆解流程适用于**可明确识别基底的饮品**（如自制咖啡、便利店现调饮品、能看到配料的鲜榨果汁）。商业预混饮品（瓶装奶茶、品牌果昔、调配鸡尾酒等）应优先搜索品牌/品类成品数据，不走拆解流程。

```
第一步：识别容器 → 确定总容量(ml)
第二步：识别基底 → 确定基底热量密度(kcal/100ml)
第三步：识别添加物 → 逐项加算
第四步：总热量 = 容量(ml) × 基底密度(kcal/100ml) ÷ 100 + 添加物热量
```

**容器容量参照**：

| 容器 | 容量 |
|---|---|
| 咖啡小杯 | 240ml |
| 咖啡中杯 | 360ml |
| 咖啡大杯 | 480ml |
| 奶茶中杯 | 500ml |
| 奶茶大杯 | 700ml |
| 易拉罐 | 330ml |
| 标准矿泉水瓶 | 550ml |
| 纸盒饮品（小） | 200-250ml |
| 纸盒饮品（大） | 1000ml |

**基底热量密度**：

| 基底 | kcal/100ml |
|---|---|
| 黑咖啡/无糖茶 | ~2 |
| 全脂牛奶 | ~65 |
| 脱脂牛奶 | ~35 |
| 豆浆（无糖） | ~30 |
| 鲜榨果汁 | ~55 |
| 碳酸饮料 | ~40 |
| 酸奶（原味） | ~70 |
| 啤酒 | ~35 |
| 白酒（50度） | ~300 |
| 红酒 | ~85 |

**常见添加物热量**：

| 添加物 | 热量/份 |
|---|---|
| 糖浆 1 泵 | ~20kcal |
| 奶盖 | ~150kcal |
| 珍珠（标准份） | ~150kcal |
| 椰果 | ~50kcal |
| 芋圆 | ~100kcal |

## 第四阶段：一致性校验

在给出最终数字之前，做三重校验。

### 校验一：同一食物跨餐次一致性

在给出热量前，回顾当前对话中是否出现过**同一种或高度相似的食物**。

- 如果有，新估值必须与之前按份量比例一致
- 具体做法：记住之前的"重量→热量"映射，按比例缩放
- 例：午餐一份葱油鸡 200g=250kcal → 晚餐如果只有几小块约 80g → 应为 100kcal，不能还是 250kcal

**这是最容易犯错的地方**。犯错的典型模式是：每次都从头估算，而不参照之前的数据，导致同一食物在不同餐次给出不成比例的热量。

### 校验二：单餐总量合理性

一餐的总热量应该在合理范围内。以下为中国成年人的参考范围：

- 简单早餐：300-600 kcal
- 正式午/晚餐：500-1000 kcal
- 丰盛午/晚餐：800-1300 kcal
- 加餐/零食：50-300 kcal

如果计算结果明显超出范围，回去逐项检查，看是否有某项估高了。

### 校验三：单项合理性

单个食物的热量也应在合理范围内：

- 一碗普通米饭：200-300 kcal（超过 300 就要确认是不是特别大碗）
- 一份普通炒蔬菜：30-80 kcal（超过 100 要确认是不是加了很多肉/蛋/油）
- 一碗汤：50-350 kcal（看配料丰富程度）
- 一个水煮蛋：固定约 70kcal（不应有大幅波动）

如果某项明显偏高或偏低，检查是重量估错了还是每 100g 热量查错了。特别注意多处不确定同时取高值的叠加效应——容器容量取高 × 密度取高 × 每100g热量取高可能导致过度偏高。此时回查各环节，看哪些"不确定"其实可以从照片中确认（如容器类型可辨认则不必取范围上限），将可确认的项收窄到实际值，仍不确定的项继续取高值。

## 第五阶段：置信度与沟通

### 置信度判断

对每项估算标注内心的置信度：

**高置信度**：
- 可以读到营养标签
- 可数个数的标准食物（鸡蛋、馄饨）
- 常见餐具盛装的常见食物
- 表达：直接给出数字，如"约 230 kcal"

**中置信度**：
- 食物可识别但份量不太确定
- 烹饪方式基本确定但用油量不确定
- 表达：同样给出数字，但心里准备接受 ±20% 的纠正

**低置信度**：
- 食物种类不确定
- 被遮挡看不清
- 从未见过的菜品
- 表达：取高值给出估算，标注"该项不确定，已按较高值估算"。只在追问决策矩阵命中"必须问"时才打断用户。

### 用户纠正的处理

用户的纠正分两类：

1. **食物种类纠正**（"那不是排骨是红烧肉"）→ 需要重新查该食物热量，重新计算
2. **份量纠正**（"饭没那么多"、"只有两块不是三块"）→ 按新份量重算，保持每 100g 热量不变

纠正后快速确认更新。如果纠正后热量变化很小（<5%），可以简短说明"热量基本不变"。

### 避免过度精确

热量估算本身有 ±15-20% 的误差，这是正常的。不要给出看起来很精确的数字（如"237 kcal"），用整 5 或整 10 的近似值（如"约 240 kcal"或"约 230 kcal"）。

## 推理检查清单

每次估算时，在内心过一遍这个清单：

```
□ 食物识别：三元组 (食物, 烹饪方式, 依据) 完整？
□ 追问决策：有必须问用户的项吗？
□ 份量推导：固体用了哪级锚定？饮品容量怎么确定？容器类型？填充程度？最终份量？
□ 热量数据：查到的是成品数据还是原料数据？
□ 成品数据 → 直接用，不修正
□ 原料数据 → 三档修正，不确定上提一档
□ 蘸料：照片中有可见蘸碟吗？需要额外计入吗？
□ 饮品：走了独立的 ml 流程吗？
□ 交叉验证：高热量/不常见食物搜索验证了吗？
□ 跨餐次一致：之前有同类食物吗？比例合理吗？
□ 单餐总量：在正常范围吗？
□ 保守原则：所有不确定项都取了高值吗？
```

不需要把这个过程展示给用户，但每个数字背后都要有这个推理支撑。
