# 食物热量估算方法论

## 核心原则

本 skill 不硬编码任何热量数值。所有热量数据应在估算时通过以下优先级获取：

1. **包装营养标签**（用户拍了标签） → 直接读取，最准确
2. **品牌官方数据**（如瑞幸贴纸上标注的热量） → 直接使用
3. **训练知识中的营养数据**（中国食物成分表、USDA等） → 按推理流程计算
4. **Web 搜索**（以上都不确定时） → 搜索具体食物的热量数据

绝不凭直觉直接给出一个总热量数字。每个数字都要有推导过程。

---

## 第一阶段：食物识别

### 扫描顺序

拿到一张食物照片后，按以下顺序系统扫描，避免遗漏：

1. **主食**（米饭、面、馒头、饼、年糕）— 通常体积最大、颜色浅
2. **蛋白质主菜**（肉、鱼、蛋、豆制品）— 盘中最显眼的部分
3. **蔬菜/配菜** — 绿色或彩色部分
4. **汤类** — 碗中有汤水的食物，注意辨认配料
5. **饮品** — 牛奶盒、咖啡杯、饮料瓶罐
6. **零食/包装食品** — 独立包装物

### 识别关键点

**看文字**：包装上的品牌名、品名、口味、容量、营养标签都是线索。饮品杯上的贴纸通常包含名称、杯型、甜度、定制选项。

**看烹饪方式**：同一食材不同做法热量差异极大，从外观判断：
- 表面油亮、有焦痕 → 煎/炸/炒
- 颜色清淡、有汤水 → 蒸/煮/烫/炖
- 深色酱汁裹覆 → 红烧/卤/酱
- 表面干燥、有烤痕 → 烤/烘

**不确定就问**：肉类品种（猪/牛/鸡/鸭）、相似蔬菜（菜苔/菠菜/油菜）、汤的配料被遮挡等情况，直接让用户确认，不要猜。

**不需要问的情况**：热量差异很小的变体（炒油菜 vs 炒小白菜）、调味差异（咸淡不影响热量）、正常范围内的微小份量浮动。

## 第二阶段：重量估算

这是最关键的一步。**永远不要跳过重量直接给热量**，必须先有重量估算，再用重量算热量。

### 方法一：餐具参照法

通过餐具推算容量，再通过容量推算重量。

**步骤**：
1. 判断餐具类型（饭碗/汤碗/盘子/外卖盒）
2. 估算餐具尺寸（通过与筷子、勺子、手、手机的对比。标准筷子长约24cm，成人手掌宽约8-9cm）
3. 估算餐具容量（口径11-12cm的碗约300-400ml，13-15cm的碗约500-600ml，18-20cm的盘子面积约250-300cm²）
4. 观察填充程度（满/八分满/半碗/浅浅一层）
5. 容量 × 填充比例 × 食物密度 = 重量

**食物密度经验**：
- 米饭：约0.6-0.8g/ml（松散到压实）
- 面条（熟）：约0.7-0.9g/ml
- 蔬菜（炒/烫）：约0.3-0.5g/ml（叶菜疏松）
- 肉类（块/片）：约0.8-1.0g/ml（密实）
- 汤类：约1.0g/ml（接近水）

### 方法二：数量计数法

可以数个数的食物，逐个计数再查单个重量。

**步骤**：
1. 数清个数
2. 回忆或搜索该食物的单个标准重量（如鸡蛋去壳约50g、鹌鹑蛋去壳约8g、馄饨约25-35g/个）
3. 个数 × 单个重量 = 总重量

这是最准确的估算方法之一，优先用于可数食物。

### 方法三：面积厚度法

盘中平铺的食物（如切片肉、炒菜），用面积×厚度估算体积。

**步骤**：
1. 估算食物覆盖的面积（参照盘子大小）
2. 估算平均厚度（一层叶菜约1-2cm，肉片堆约2-3cm）
3. 面积 × 厚度 = 体积
4. 体积 × 密度 = 重量

### 方法四：包装直接读取

包装食品上有净含量，直接使用，不需要估算。

**步骤**：
1. 找到"净含量"标注（如"50g"、"250ml"）
2. 对于液体，注意 ml 和 g 的区别（牛奶约1.03g/ml，可近似为1:1）

### 方法五：份量比例法

同一食物在不同餐次出现时，通过与之前的视觉对比估算份量。

**步骤**：
1. 回忆之前同一食物的照片和估算重量
2. 对比当前照片中的盛装量
3. 按比例缩放重量

这是保持一致性的关键方法，详见第四阶段。

### 同一碗/盘多种食物的拆分

一个容器中有多种食物时：
1. 先估计整个容器的食物总体积
2. 目测每种食物占据的体积比例（注意堆叠情况）
3. 分别按各自密度计算重量

## 第三阶段：热量计算

### 基本公式

```
热量(kcal) = 重量(g) × 该食物每100g热量(kcal) ÷ 100
```

### 获取"每100g热量"

按以下优先级获取：

1. **用户提供的营养标签照片** → 直接读取"每100g/100ml"对应的能量值。注意：中国标签常用"千焦(kJ)"，需除以4.184转换为kcal
2. **包装上的总热量标注** → 如瑞幸贴纸"≈267大卡"，直接使用，不需要再算
3. **训练知识** → 回忆中国食物成分表或USDA数据库中的数据
4. **Web搜索** → 搜索"[食物名] 每100g 热量 kcal"

### 烹饪方式修正

如果获取的是原料基础热量（生重/水煮），需要根据实际烹饪方式修正：

| 烹饪方式 | 修正倍数 | 原因 |
|----------|---------|------|
| 水煮/蒸/烫/白灼 | ×1.0 | 不额外加油脂 |
| 清炒（少油） | ×1.1-1.2 | 少量油 |
| 家常炒 | ×1.2-1.3 | 正常油量 |
| 红烧/卤 | ×1.1-1.2 | 糖+酱油+少量油 |
| 煎 | ×1.3-1.5 | 底部持续接触油 |
| 油炸/干炸 | ×1.5-2.0 | 大量吸油 |

**注意**：如果查到的数据已经是成品数据（如"红烧肉 每100g 250kcal"），不要重复修正。只有查到原料数据时才需要修正。

### 汤类特殊处理

汤的热量不能简单按总重量算，因为大部分重量是水（几乎0热量）。

**正确做法**：
1. 分别识别汤中的固体配料（肉、蛋、豆腐、蔬菜等）
2. 估算每种固体配料的重量
3. 分别查每种配料的热量
4. 汤水部分粗估：清汤 5-15kcal/100ml，浓汤/奶汤 20-40kcal/100ml
5. 固体热量 + 汤水热量 = 总热量

### 复合菜品处理

多种食材混合的菜品（如咖喱鸡肉配土豆胡萝卜）：

**方法A**（拆分法）：估算总重量 → 目测各食材比例 → 分别算热量 → 求和
**方法B**（整体法）：直接搜索该菜品的综合热量数据 → 乘以估算重量

两种方法都可以，建议在心里都算一遍看是否一致。

## 第四阶段：一致性校验

在给出最终数字之前，做三重校验。

### 校验一：同一食物跨餐次一致性

在给出热量前，回顾当前对话中是否出现过**同一种或高度相似的食物**。

- 如果有，新估值必须与之前按份量比例一致
- 具体做法：记住之前的"重量→热量"映射，按比例缩放
- 例：午餐一份葱油鸡200g=250kcal → 晚餐如果只有几小块约80g → 应为100kcal，不能还是250kcal

**这是最容易犯错的地方**。犯错的典型模式是：每次都从头估算，而不参照之前的数据，导致同一食物在不同餐次给出不成比例的热量。

### 校验二：单餐总量合理性

一餐的总热量应该在合理范围内。以下为中国成年人的参考范围：

- 简单早餐：300-600 kcal
- 正式午/晚餐：500-1000 kcal
- 丰盛午/晚餐：800-1300 kcal
- 加餐/零食：50-300 kcal

如果计算结果明显超出范围，回去逐项检查，看是否有某项估高了。

### 校验三：单项合理性

单个食物的热量也应在合理范围内：

- 一碗普通米饭：200-300 kcal（超过300就要确认是不是特别大碗）
- 一份普通炒蔬菜：30-80 kcal（超过100要确认是不是加了很多肉/蛋/油）
- 一碗汤：50-350 kcal（看配料丰富程度）
- 一个水煮蛋：固定约70kcal（不应有大幅波动）

如果某项明显偏高或偏低，检查是重量估错了还是每100g热量查错了。

## 第五阶段：置信度与沟通

### 置信度判断

对每项估算标注内心的置信度：

**高置信度**：
- 可以读到营养标签
- 可数个数的标准食物（鸡蛋、馄饨）
- 常见餐具盛装的常见食物
- 表达：直接给出数字，如"约 230 kcal"

**中置信度**：
- 食物可识别但份量不太确定
- 烹饪方式基本确定但用油量不确定
- 表达：同样给出数字，但心里准备接受 ±20% 的纠正

**低置信度**：
- 食物种类不确定
- 被遮挡看不清
- 从未见过的菜品
- 表达：主动告知不确定，给出最可能的推测并请用户确认

### 用户纠正的处理

用户的纠正分两类：

1. **食物种类纠正**（"那不是排骨是红烧肉"）→ 需要重新查该食物热量，重新计算
2. **份量纠正**（"饭没那么多"、"只有两块不是三块"）→ 按新份量重算，保持每100g热量不变

纠正后快速确认更新。如果纠正后热量变化很小（<5%），可以简短说明"热量基本不变"。

### 避免过度精确

热量估算本身有 ±15-20% 的误差，这是正常的。不要给出看起来很精确的数字（如"237 kcal"），用整5或整10的近似值（如"约 240 kcal"或"约 230 kcal"）。

## 推理检查清单

每次估算时，在内心过一遍这个清单：

```
□ 食物识别：这是什么？烹饪方式是什么？
□ 重量推导：用了什么参照物？估算重量是多少克？
□ 热量数据来源：标签/训练知识/搜索？每100g多少kcal？
□ 热量计算：重量 × 每100g热量 ÷ 100 = ?
□ 烹饪修正：需要吗？修正后是多少？
□ 跨餐次一致：之前有同类食物吗？比例合理吗？
□ 单餐总量合理：这一餐加起来在正常范围吗？
□ 置信度：高/中/低？需要问用户确认吗？
```

不需要把这个过程展示给用户，但每个数字背后都要有这个推理支撑。
